Problem Statement
"

Elderly users aged 60 and above, particularly those with low digital literacy, struggle significantly to use modern banking apps. Complex multi-step interfaces, small fonts, confusing icons, and the complete absence of guided assistance create an environment that is hostile to this population — resulting in financial dependency on others, heightened vulnerability to fraud, and a deep sense of anxiety and exclusion from essential financial services.

This is a problem of design failure — not a failure of the user. The system was never built with them in mind.

A Personal Lens

Why This Problem Matters

This isn't an abstract research question. It comes from real, observed frustration.

1

The Hesitation

A parent or grandparent picks up the phone to transfer money — and pauses. They're afraid of pressing the wrong button, sending to the wrong person, or losing money entirely.

2

The Ask for Help

Instead of completing the task independently, they wait for a family member to "do it for them." Every time. This erodes confidence and creates dependency.

3

The Root Cause

The app was designed for a younger, tech-savvy audience. This group was never considered — and it shows in every interaction.

Analysis

Where Apps Fall Short

These aren't minor UX oversights — they are systemic barriers that prevent a significant portion of users from completing basic financial tasks independently.

Tiny Typography

Font sizes below 14px are unreadable for ageing eyes

Information Overload

Too many options on a single screen

No Onboarding

Users left to figure it out alone

Small Tap Targets

Buttons too small for imprecise motor control

Banking Jargon

"IMPS", "NEFT" mean nothing to many users

No Error Recovery

Mistakes feel permanent and costly

Each pain point compounds the others. A user with poor vision who also struggles with jargon and receives no voice guidance faces compounding failure at every step.

Three Dimensions

Accessibility Barriers

Visual Barriers

  • Font sizes too small for ageing eyes
  • Low contrast between text and background
  • Icon-only navigation with no text labels

Cognitive Barriers

  • Multi-step flows with no progress indicators
  • Overloaded screens competing for attention
  • Banking terminology without plain-language alternatives

Interaction Barriers

  • Tap targets far too small for imprecise motor control
  • Complex gesture-based navigation (swipe, pinch, long-press)
  • No haptic or audio feedback on actions

Features

Key Design Features

Large Buttons & Typography

Minimum 16px body text, 48px tap targets — designed for ageing eyes and imprecise touch

Clear Step Instructions

Every screen tells the user exactly what to do in plain, simple language

Error Prevention Prompts

Warnings before irreversible actions — catching mistakes before they happen

Confirmation Screen

Full summary before payment execution — name, amount, method — all clearly displayed

Optional Voice Feedback

Audio read-aloud of amounts, names, and instructions — always optional, always available

Progress Indicators

"Step 2 of 3" — users always know where they are and how much is left

Non-Negotiables

Requirements Addressed

Accessibility

Large touch targets (48px minimum), high-contrast text (WCAG AA), readable typography (16px+ body), multimodal output (text + voice), and support for screen readers.

✓ ADDRESSED

Clarity

One action per screen, plain-language instructions (no jargon), clear visual hierarchy, progress indicators showing position in flow, and consistent component patterns throughout.

✓ ADDRESSED

Error Prevention

Confirmation screen before every transaction, warning prompts for irreversible actions, input validation with clear feedback, and ability to go back at any step without losing data.

Reflection

Personal Learnings

Biggest Challenge

Understanding real user struggles beyond assumptions. It's easy to think "just make the font bigger" — but the real challenge is understanding the cumulative anxiety that builds when every single interaction feels uncertain. Each small confusion compounds into a complete refusal to try.

What I'd Improve

User testing with real elderly users. Research and heuristic analysis can only go so far — watching an actual 65-year-old try to complete a transaction would reveal blind spots that no amount of theory can predict. This is the critical next step for validation.

Key Learning

Inclusive design is not adding features — it's removing barriers.
